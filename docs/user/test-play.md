# テストプレイ

エディタ内で、編集中のプロジェクトをそのままプレイできます。書き出し前の動作確認に使います。

## 起動

1. **Ctrl+4** でテストプレイタブを開く
2. **テストプレイ開始** をクリック
3. 埋め込み iframe 内でランタイムが起動する

## 章ありプロジェクトの流れ

サンプルのように章が定義されている場合:

1. タイトル画面
2. 拠点（編成・ショップ）
3. 出撃 → 戦闘マップ
4. 勝利または敗北

## デバッグオプション

| オプション | 効果 |
|------------|------|
| デバッグシード | 戦闘の乱数を固定し、同じ操作で同じ結果を再現 |
| 無敵 | プレイヤーユニットがダメージを受けない |

バランス調整の途中確認や、イベントの分岐テストに便利です。公開用の書き出しには無敵は含まれません。

## 確認すべき項目

- [ ] 勝利条件・敗北条件が意図どおり発火する
- [ ] 章開始・増援イベントのタイミング
- [ ] BGM / SE の切り替え
- [ ] 拠点の編成上限（`maxDeploy`）
- [ ] 支援会話（該当する場合）

## テストプレイと書き出しの違い

ロジックは同じランタイム（`packages/runtime`）を使いますが、表示領域は iframe 内に限定されます。書き出し成果物ではフルウィンドウで動作します。最終確認は [書き出しと配布](export-and-publish.md) の手順で行ってください。

## うまく動かないとき

- **未保存** の変更はテストプレイに反映されますが、念のため保存してから再試行
- マップにユニットを置いたが DB に定義がない → データベースタブで ID を確認
- イベントでエラー → コマンドのパラメータが [event-commands.md](../event-commands.md) と一致しているか確認
