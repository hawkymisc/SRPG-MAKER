# 作成者向けドキュメント

**作成者** = SRPG を制作する人（プログラマーでなくても可）。プレイヤー向けマニュアルではありません。

## このツールでできること

- マップ（地形・ユニット配置）の編集
- ユニット・武器・スキルなどのデータベース編集
- 戦闘前後のイベント（会話・増援・BGM など）
- 章と拠点（編成・出撃）の管理
- エディタ内テストプレイ
- HTML5 / デスクトップ / モバイル向けへの書き出し

## 読む順番（初めての方）

1. [はじめに](getting-started.md) — 起動とサンプルプロジェクト
2. [制作の流れ](workflow.md) — 一通りの作業順

## クイックリファレンス

| 操作 | 方法 |
|------|------|
| タブ切替 | **Ctrl+1〜6**（プロジェクト / DB / マップ / テストプレイ / イベント / 素材） |
| 新規プロジェクト | プロジェクトタブ → **新規（サンプル）** |
| 保存 | プロジェクトタブ → **保存**（JSON またはフォルダ） |
| テストプレイ | **Ctrl+4** → **テストプレイ開始** |
| HTML5 書き出し | プロジェクトタブ → **HTML5書き出し** |

## データの置き場所

プロジェクトはフォルダ形式（Git 管理しやすい JSON 群）で保存します。構造は [プロジェクトデータ](project-format.md) を参照してください。
