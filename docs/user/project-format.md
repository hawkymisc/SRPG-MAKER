# プロジェクトデータ

プロジェクトは **JSON** で表現されます。小規模なら単一ファイル、本格的な制作では **フォルダ形式**（spec 7 章）を推奨します。

## フォルダ形式の構造

```
MyGame/
├── project.json          # プロジェクト設定・schemaVersion
├── database/
│   ├── units.json
│   ├── classes.json
│   ├── weapons.json
│   ├── items.json
│   ├── skills.json
│   └── terrain.json
├── maps/
│   └── chapter01.json    # タイル・配置・勝敗条件
├── events/
│   ├── common.json
│   └── chapter01.json
├── chapters/
│   └── chapters.json     # 章定義・拠点・ショップ
├── supports/
│   └── supports.json     # 支援会話（任意）
├── plugins/
│   └── ...               # プラグイン（任意）
└── assets/
    ├── images/
    └── audio/
        ├── bgm/
        └── se/
```

同梱サンプルは `templates/sample/` にあります。

## 単一 JSON とフォルダの違い

| 形式 | 向いている用途 | 開き方 |
|------|----------------|--------|
| 単一 `.json` | 共有・バックアップ・小規模 | ブラウザ版「開く」 |
| フォルダ | Git 管理・大規模・素材多数 | デスクトップ版「開く」推奨 |

フォルダ形式では DB・マップ・イベントがファイル分割され、差分管理しやすくなります。

## スキーマバージョン

`project.json` の `schemaVersion` はツールの互換性を示します。新しいバージョンのエディタは旧データを自動マイグレーションして読み込みます。古いツールで新しいデータを開くとエラーになる場合があります。

## 自動バックアップ

ブラウザ版では保存時に直近 **5 世代** のバックアップをローカルストレージに保持します。プロジェクトタブから復元できます。重要な版は手動で別名保存してください。

## Git で管理する場合

- フォルダ形式をそのままコミット
- `assets/` の大容量バイナリは `.gitignore` や LFS の検討
- マージ競合は JSON なのでテキストエディタでも解決可能（構造を壊さないよう注意）

## 手動編集について

上級者は JSON を直接編集できますが、**Zod スキーマ**（`packages/shared/src/schemas/`）に合致している必要があります。不正な JSON は読み込み時にエラーになります。通常はエディタ経由の編集を推奨します。
