# 制作の流れ

典型的な一章付き SRPG の制作手順です。規模に応じて繰り返し・省略してください。

## 1. 企画と章設計

- マップサイズ、味方・敵の数、勝利条件（敵全滅 / ボス撃破 / ターン生存 / 拠点防衛）を決める
- 複数マップにする場合は **章** で束ねる（拠点画面→出撃の流れ）

## 2. データベース（Ctrl+2）

- **ユニット** — 名前・クラス・初期ステータス
- **クラス** — 成長率・使用可能武器
- **武器・スキル・アイテム・地形** — 戦闘に必要な定義を揃える

先にマップに置くユニット ID が DB に存在している必要があります。

## 3. マップ（Ctrl+3）

- 地形レイヤーを描画（移動コスト・防御補正は地形 DB に依存）
- **配置** で味方・敵の初期位置、装備、AI タイプを設定
- **勝利条件・敗北条件** をマップ設定で指定

## 4. イベント（Ctrl+5）

- 章開始メッセージ（`chapterStart`）
- 増援・会話・BGM 変更など
- コマンド一覧は [event-commands.md](../event-commands.md)

## 5. 素材（Ctrl+6）

- 顔グラ・BGM・SE を追加（任意）
- フォルダ保存・書き出し zip の `assets/` に同梱される

## 6. テストプレイ（Ctrl+4）

- シード値を固定すると再現しやすい（デバッグ用）
- 拠点→出撃→戦闘→勝利/敗北まで通す

詳細: [テストプレイ](test-play.md)

## 7. 保存

| 形式 | 用途 |
|------|------|
| 単一 JSON | 手軽な共有・バックアップ |
| フォルダ | Git 管理・大規模プロジェクト（デスクトップ版推奨） |

自動バックアップは直近 5 件までローカルに保持されます（ブラウザ版）。

## 8. 書き出し

- **HTML5** — ブラウザで配布（zip 内の `game/` を Web サーバで公開）
- **Electron** — Win / macOS / Linux 向けスタンドアロン
- **モバイル** — Capacitor 用 zip（実機ビルドは別途 Android Studio / Xcode）

詳細: [書き出しと配布](export-and-publish.md)
