# ADR 0009: クラウド保存の方針（フェーズ3）

## コンテキスト

spec.md 2.3 のフェーズ3には「クラウド保存」が含まれる。
現状のプロジェクトは Git 管理可能な **フォルダ形式 JSON**（`splitProject`）であり、書き出しも zip（HTML5/Electron/Capacitor）で成立している。

フェーズ3段階では、まず **データ構造・同期単位・衝突解決** を明確化してから実装する必要がある。

## 決定（まずはローカル抽象化から段階的に）

1. **保存単位**: 「プロジェクトフォルダ（splitProject + assets）」を最小単位として扱う。
2. **抽象化**: editor 側に「ストレージアダプタ（local / cloud）」境界を設け、UIは保存先を切り替えられるようにする。
3. **クラウド実体**: MVP は “クラウドの実装先” を固定しない（GitHub / S3 / Firebase などは将来選定）。まずは adapter で差し替え可能にする。
4. **整合性**: 当面は「最後に保存したものが勝つ（LWW）」を基本とし、同時編集の競合はフェーズ4で扱う。

## 次のアクション

- `assets/` をプロジェクトの split/merge と書き出しに含める設計（M8-T2/T5）
- ストレージアダプタ I/F を定義し、既存の LocalStorage バックアップと統合（M8-T4）
- クラウド実体の候補選定と認証方式は人間判断を仰ぐ（エスカレーション対象）
