1. はじめに

SRPGツクールとは

SRPGツクールは、シミュレーションRPG（SRPG）を制作するためのオーサリングツールです。 ブラウザベースのビジュアルエディタで、マップの作成、ユニットやクラスのデータベース管理、イベントの組み立て、そしてテストプレイまでを一貫して行えます。

ビジュアルエディタ

コーディング不要。マップもデータもイベントもGUIで作成できます。

リアルタイムテスト

編集内容をすぐにテストプレイで確認。高速な試行錯誤が可能です。

マルチプラットフォーム

HTML5・デスクトップ（Electron）・モバイルへ書き出し対応。

Git対応データ形式

プロジェクトデータはJSONフォルダ形式。チーム開発にも適しています。

必要環境

ソフトウェア 必要バージョン 備考
ブラウザ Chrome / Edge 最新版 Firefox, Safari も動作しますが推奨は Chrome 系
Node.js 22+ LTS版を推奨
pnpm 9+ corepack enable で導入可能
Git 任意 プロジェクトのバージョン管理用

インストール手順

Step 1: リポジトリのクローン

$ git clone https://github.com/your-org/srpg-maker.git $ cd srpg-maker

Step 2: 依存パッケージのインストール

$ pnpm install
pnpmが未インストールの場合は corepack enable を先に実行してください。Node.js 22 には corepack が同梱されています。

起動方法

ブラウザ版（推奨）

$ pnpm dev:editor # エディタが起動します → http://localhost:5173

デスクトップ版（Electron）

# ターミナル1: エディタサーバー $ pnpm dev:editor # ターミナル2: Electronアプリ $ pnpm dev:editor:electron

テストプレイ用ランタイム

$ pnpm dev:runtime # ゲームランタイムが起動します → http://localhost:5174
テストプレイを行うには、エディタとは別にランタイムサーバーも起動する必要があります。

3. プロジェクトタブ

プロジェクトタブでは、ゲーム全体の構成を管理します。章（シナリオ）の作成と順序管理、保存と書き出しを行います。

サンプルプロジェクトを読み込む

初めて使う場合は、サンプルプロジェクトを読み込んで動作を確認しましょう。

  1. エディタを起動し、プロジェクトタブ（Ctrl + 1）を開きます
  2. 画面上部の 「新規（サンプル）」 ボタンをクリックします
  3. サンプルデータが読み込まれ、章・ユニット・マップが自動的にセットアップされます
サンプルプロジェクトには、チュートリアルで使用するすべてのデータが含まれています。まずはこれをベースに各機能を学びましょう。

章（シナリオ）管理パネル

SRPGツクールでは、ゲーム内のステージを「章（Chapter）」として管理します。各章には1つのマップが紐付き、章の順序がゲームの進行順序になります。

章リスト + 章を追加
第1章: 王国の危機 草原マップ ⠿
第2章: 森の戦い 森林マップ ⠿
第3章: 最終決戦 城マップ ⠿

操作方法

  • 章名の編集 — 章名をダブルクリックすると、テキストを直接編集できます。
  • マップの紐付け — 各章の右側にあるマップ名をクリックすると、紐付けるマップを選択できます。
  • 開始章の指定 — 左端のラジオボタンで、ゲーム開始時に最初にプレイされる章を指定します。
  • 並び順の変更 — 右端のドラッグハンドル（⠿）をドラッグして、章の順序を変更できます。

保存方法

作成したプロジェクトの保存方法は、実行環境によって異なります。

Browser

ブラウザ版

プロジェクト全体を1つのJSONファイルとしてダウンロードします。次回はファイルを読み込んで再開できます。

Desktop

デスクトップ版

フォルダ形式で保存されます。各データが個別のJSONファイルになるため、Gitでのバージョン管理に適しています。

Auto

自動バックアップ

5世代分の自動バックアップが保持されます。予期しないデータ消失に備える安心機能です。

書き出し（エクスポート）

完成したゲームは、複数のプラットフォーム向けに書き出すことができます。

出力形式 説明 用途
HTML5 Webブラウザで動作するゲームを出力 Webサイトへの公開、itch.io等
Electron Windows / macOS / Linux用の実行ファイルを出力 ダウンロード配布
モバイル Capacitor経由でiOS / Android向けにビルド アプリストア配信
書き出し機能の詳細設定については、セクション8「素材・書き出し・Tips」で解説します。

4-2. クラス

ユニットの職業（兵種）を定義します。クラスによって移動タイプや使用可能武器が決まります。

基本設定

  • 名前 — クラスの表示名（例: ロード、兵士、魔道士）
  • 移動タイプ — 地形との相性を決定（下表参照）
  • 基準能力値 — クラスが持つベース補正値
  • 使用可能武器種 — 剣 / 槍 / 斧 / 弓 / 魔法から複数選択可
  • 能力上限 — 各パラメータの最大値（任意設定）
  • 転職先クラス — クラスチェンジ時の遷移先

移動タイプの効果

移動タイプ特徴地形影響
歩兵標準的な移動森・山で移動コスト増加
騎兵平地で高速移動森で大幅コスト増、山は進入不可
飛行地形無視で移動全地形コスト1（弓に弱い）
重装高防御・低移動森・山で移動コスト増加

4-6. 地形

マップタイルの地形効果を定義します。地形はマップエディタで配置するタイルに紐づきます。

地形パラメータ

  • 移動コスト — 移動タイプ別の消費移動力
  • 回避補正 — この地形に居るユニットの回避率ボーナス
  • 防御補正 — この地形に居るユニットの被ダメージ軽減
  • ターン回復量 — ターン終了時のHP回復（砦など）
  • 進入不可フラグ — 特定の移動タイプが進入できない設定

戦闘計算式の概要

データベースで設定した値がどのように戦闘に反映されるか、計算式をまとめます。

攻撃力 = 力（or 魔力） + 武器威力 + 三すくみ補正
ダメージ = max(0, 攻撃力 - 守備（or 魔防） - 地形防御補正)
命中率 = (技×2 + 幸運/2 + 武器命中 + 三すくみ命中)
− (相手速さ×2 + 相手幸運 + 地形回避)
必殺率 = 技/2 + 武器必殺 − 相手幸運
追撃条件: 速さ差 ≥ 4 で追撃発生（2回攻撃）
必殺ダメージ = 通常ダメージ × 3
計算式の詳細やカスタマイズについては仕様書（docs/spec.md 5.2章）を参照してください。 ここに記載しているのは既定値です。

5. マップエディタ

マップタブ（Ctrl + 3）ではゲームの戦場を作成します。地形を描画し、ユニットを配置し、勝敗条件を設定します。

マップの基本操作

マップの選択

ツールバー左端のドロップダウンから、編集対象のマップを選択します。マップはデータベースで事前に定義しておく必要があります。

描画ツール

3種類のツールを切り替えて地形を描画します。

ツール 操作 用途
ペンツール セルをクリック / ドラッグ 1セルずつ地形を塗る。細かい調整に最適
矩形ツール 始点クリック → 終点クリック 2点で矩形範囲をまとめて塗りつぶし
塗りつぶしツール セルをクリック 同じ地形タイプの連結領域を一括で塗り替え

レイヤー

マップは 2層構造 になっています。ツールバーのレイヤードロップダウンで切り替えます。

  • bottom（下層） — 地面の基本地形を配置します（平原、水、山など）
  • top（上層） — 下層の上に重ねて描画するオブジェクトを配置します（森、砦など）

地形パレット

画面左のパレットから地形を選択します。各地形にはゲーム内で効果（回避率ボーナスなど）が設定されています。

ユニット配置

地形を描画したら、次はユニットをマップ上に配置します。ツールバーの「配置」ボタンでユニット配置モードに切り替えます。

配置の手順

  1. ユニット選択 — パレットからデータベースに定義済みのユニットを選びます
  2. 陣営選択 — ユニットの所属陣営を指定します
    • プレイヤー（青）— プレイヤーが操作する自軍ユニット
    • 敵（赤）— AIが操作する敵軍ユニット
    • 第三勢力（黄）— 独自行動をとる中立ユニット
  3. AI選択 — 敵・第三勢力ユニットの行動パターンを指定します（下表参照）
  4. 配置 — マップ上のセルをクリックしてユニットを配置
  5. 削除 — Shift + クリックで配置済みユニットを除去

AIタイプ一覧

AIタイプ 内部名 行動パターン
突撃 charge 最も近いプレイヤーユニットに向かって移動・攻撃する。積極的な敵に最適
待ち伏せ ambush 初期位置に留まり、攻撃範囲に入った敵のみ攻撃する。一定範囲内に入ると突撃に切替
守備 guard 初期位置周辺から離れず防衛。拠点や要所を守らせるのに使用
移動のみ move_only 攻撃せず移動のみ。NPC・逃走ユニット・イベント用のキャラクターに使用
プレイヤー陣営のユニットにはAI設定は不要です（プレイヤーが直接操作します）。

勝利条件・敗北条件

各マップにはクリア判定のための勝利条件と、ゲームオーバーとなる敗北条件を設定します。

勝利条件（4種）

タイプ 内部名 説明
敵全滅 defeat_all_enemies マップ上の敵軍ユニットをすべて撃破するとクリア
ボス撃破 defeat_boss 指定したボスユニットを撃破するとクリア。ボスIDを指定する
生存 survive_turns 指定ターン数だけ生存（全滅せず耐える）するとクリア。防衛戦向け
拠点防衛 defend_point 指定座標を敵に占領されずに守り切るとクリア。座標を指定する

敗北条件

  • 自軍全滅 — プレイヤー陣営のユニットがすべて撃破された（デフォルトON）
  • ターン上限超過 — 指定ターン数を超えてもクリアできなかった場合（オプション）
勝利条件は章ごとに1つ設定します。ゲーム中にイベントの CHANGE_OBJECTIVE コマンドで動的に差し替えることも可能です（セクション6参照）。

増援の設定

マップの途中で追加ユニットを出現させる「増援」を設定できます。画面左パレット下部の「増援」セクションから管理します。

増援パラメータ

6. イベント

イベントタブ（Ctrl + 5）では、マップ上で起こる出来事をコマンド列として記述します。会話、増援、BGM変更、勝敗条件の切り替えなどを制御できます。

トリガー（7種）

イベントがいつ発火するかを決める条件です。1つのイベントに1つのトリガーを設定します。

トリガー 内部名 発火タイミング
章開始 chapterStart 章の開始時に1回だけ発火。オープニング会話や初期演出に使用
章クリア chapterEnd 章のクリア時に発火。エンディング演出や次章への導入に使用
ターン開始 turnStart 指定ターンの開始時に発火。ターン番号を指定する
ユニット撃破 unitDefeated 指定ユニットが撃破された時に発火。ユニットIDを指定する
マス到達 tileReached 指定マスにユニットが到達した時に発火。座標を指定する
会話 talk 隣接したユニット同士が「会話」コマンドを実行した時に発火
宝箱 chestOpened 宝箱が配置されたマスを調べた時に発火

コマンド一覧（15種・凍結）

トリガーで発火したイベントは、以下のコマンドを上から順に実行します。15種で仕様が凍結されています。

メッセージ系

コマンド パラメータ 説明
SHOW_MESSAGE 話者ID、顔グラID、テキスト メッセージウィンドウにテキストを表示する
SHOW_CHOICES 選択肢（2〜6個）、格納先変数 プレイヤーに選択肢を表示。選択した番号（index）を変数に格納する

変数・条件系

コマンド パラメータ 説明
SET_VARIABLE 変数名、演算子（=, +, -, ×）、値 変数を操作する。フラグ管理や計算に使用
SET_SWITCH スイッチ名、操作（ON / OFF / トグル） スイッチ（ON/OFF値）を切り替える
BRANCH 条件式、then節、else節 条件分岐。変数 / スイッチ / ユニット生存 で分岐。ネスト最大5段

ユニット操作系

コマンド パラメータ 説明
SPAWN_UNIT ユニットID、座標、陣営、AI ユニットをマップ上に出現させる
REMOVE_UNIT ユニットID、演出（フェード / ワープ / なし） ユニットをマップから離脱させる
MOVE_UNIT ユニットID、目的座標 ユニットを指定マスへ移動演出つきで動かす

演出系

コマンド パラメータ 説明

SCREEN_EFFECT エフェクト種別、パラメータ 画面効果（フェードイン / フェードアウト / シェイク / ティント）
WAIT ミリ秒 指定時間だけ待機する

ゲーム進行系

コマンド パラメータ 説明

イベントの作り方

以下のステップでイベントを作成します。

Step 1: イベントタブを開く

Ctrl + 5 でイベントタブに切り替えます。

Step 2: イベントを新規作成

画面上部の「追加」ボタンをクリックして、新しいイベントを作成します。

Step 3: トリガーを選択

トリガードロップダウンから発火条件を選びます（例: chapterStart）。

Step 4: コマンドを追加

コマンドの種別をドロップダウンから選び、「追加」ボタンでコマンドリストに追加します。

Step 5: パラメータを編集

追加したコマンドをクリックして、右パネルでパラメータ（テキスト、座標、IDなど）を編集します。

Step 6: 順序の変更・削除

↑↓ボタンでコマンドの実行順序を変更、×ボタンで削除できます。

よくあるイベントパターン

パターン1: 章開始メッセージ

章が始まった時にストーリー演出を挿入する基本パターン。

トリガー chapterStart
❶ SHOW_MESSAGE — 「敵が攻めてきた！迎え撃て！」

パターン2: ターン3で増援

指定ターンに増援ユニットを登場させるパターン。

トリガー turnStart （ターン3）
❶ SHOW_MESSAGE — 「援軍が到着した！」
❷ SPAWN_UNIT — 味方ユニットを (2, 8) に配置

パターン3: ボス撃破で勝利条件変更

ボスを倒した後に勝利条件を変更して新たな目標を提示するパターン。

トリガー unitDefeated （ボスID指定）
❶ CHANGE_OBJECTIVE — 勝利条件を「生存5ターン」に変更
❷ SHOW_MESSAGE — 「敵の増援が来る！5ターン耐えろ！」

マップとイベントの紐付け

作成したイベントは、マップタブ（Ctrl + 3）側で対象マップに紐付ける必要があります。

  1. マップタブを開き、対象のマップを選択します
  2. 左パネル下部の「イベント」セクションを開きます
  3. attachボタンで使用するイベントIDを紐付け、detachボタンで解除できます

1つのマップに複数のイベントを紐付けることができます。同一トリガーのイベントが複数ある場合、リスト順に実行されます。

BRANCH コマンドでネスト（入れ子）を使うと複雑な条件分岐が作れますが、ネストは最大5段までです。それ以上の深さが必要な場合は、スイッチや変数を活用してロジックを分割しましょう。
イベントコマンドは15種で仕様凍結されています。これ以上のコマンドはプラグインAPI（フェーズ2）で拡張予定です。

7. テストプレイ

テストプレイタブ（Ctrl+4）で、編集中のゲームをすぐにプレイして動作確認ができます。

テストプレイの起動方法

Step 1 ランタイムサーバーを起動（別ターミナルで）

pnpm dev:runtime

Step 2 テストプレイタブ（Ctrl+4）を開く

Step 3「テストプレイ開始」ボタンをクリック

Step 4 エディタ内のiframeでゲームが起動します

※「別ウィンドウで起動」を選ぶと独立したウィンドウでプレイできます

章ありプロジェクトの流れ

タイトル画面 → 拠点（編成・ショップ） → 出撃 → 戦闘マップ → 勝利 → 次章へ 敗北 → リトライ

デバッグオプション

オプション効果
デバッグシード 戦闘の乱数を固定（デフォルト: 42001）。同じ操作で同じ結果を再現できる
無敵モード プレイヤーユニットがダメージを受けない。バランス調整の途中確認に便利

テストプレイで確認すべきチェックリスト

うまく動かないとき

症状対処
何も表示されない pnpm dev:runtime が起動しているか確認
ユニットが表示されない データベースにユニット定義があるか確認
イベントが発火しない マップにイベントが紐付けされているか確認
ダメージが0 武器の威力や装備を確認
Tip: テストプレイ内のデータは保存されません。バランス調整にはシード値を固定して同じ戦闘を再現すると効率的です。

8. 素材・書き出し・Tips

ゲームに使う画像や音楽の管理方法、完成したゲームの配布方法、そして制作に役立つTipsを紹介します。

素材管理

素材タブ（Ctrl+6）

  • 画像を追加 → assets/images/
  • BGMを追加 → assets/audio/bgm/
  • SEを追加 → assets/audio/se/

一覧からプレビュー（画像のみ）や削除が可能です。

推奨形式

種別形式備考
タイル・ユニット画像 PNG 透過対応。タイルサイズ（32/48/64px）に合わせる
顔グラフィック PNG 会話ウィンドウ用
BGM / SE OGG or MP3 ブラウザ互換優先ならOGG

イベント・DBとの紐付け

  • 会話の顔グラ → SHOW_MESSAGE の faceId
  • BGM → PLAY_BGM の bgmId（ファイル名ベース）
  • SE → PLAY_SE の seId

書き出しと配布

3つの書き出し形式

HTML5
  • Web配布に最適（itch.io、GitHub Pages）
  • zipを展開 → game/ をWebサーバで公開
npx serve .
python -m http.server 8080
Electron
  • デスクトップ向け実行ファイル
  • Win / macOS / Linux 対応
  • ダウンロード配布に最適
モバイル
  • Capacitor経由でiOS/Androidアプリ
  • Android Studio / Xcodeでビルドが必要

制作Tips

Tip 1: まずデータベースでユニット・武器を定義してからマップに配置しましょう。
Tip 2: テストプレイのシード値を固定すると、同じ操作で同じ結果を再現できてデバッグに便利です。
Tip 3: 章開始イベント（chapterStart + SHOW_MESSAGE）でストーリーの導入を書きましょう。
Tip 4: 増援は turnStart + SPAWN_UNIT の組み合わせが定番です。
Tip 5: 地形効果（森の回避+20、砦の防御+3・回復10）をうまく使ったマップ設計を心がけましょう。
Tip 6: 保存はこまめに！ブラウザ版は5世代のバックアップが自動保持されます。
Tip 7: 三すくみ（剣 > 斧 > 槍 > 剣）を意識した敵配置で戦略性が生まれます。

制作フロー まとめ

  1. 起動（pnpm dev:editor + pnpm dev:runtime）
  2. プロジェクト設定と章構成
  3. データベースでユニット・武器・地形を定義
  4. マップで地形描画とユニット配置
  5. イベントで会話・増援・演出を設定
  6. 素材の追加（画像・BGM・SE）
  7. テストプレイで動作確認
  8. 書き出しと配布
