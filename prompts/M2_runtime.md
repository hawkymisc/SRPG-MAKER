# M2委任プロンプト:Phaserランタイム

前提:M1完了。担当は主にruntime-dev。ルール判定をruntimeに書かないこと。

---

マイルストーンM2を実施してください。ゴールは「sharedのロジックを使い、サンプル章がブラウザで遊べて、かつヘッドレス自動プレイでクリアできる状態」です。

## タスク

1. **シーン骨格とマップ描画**(runtime-devに委任)
   Boot/Title/BattleMapシーン。テンプレートのマップJSONを読み、Tilemapで描画。カーソル移動(キーボード/マウス/タッチ)とユニットスプライト表示。

2. **移動・攻撃のUIフロー**(runtime-devに委任)
   ユニット選択 → sharedの移動範囲計算結果をハイライト → 移動 → 行動メニュー(攻撃/アイテム/待機)→ 攻撃対象選択(戦闘予測値表示:与ダメ・命中・必殺)→ resolveAction呼び出し → ログに応じたHPバー/撃破演出。

3. **敵フェイズとターン進行**(runtime-devに委任)
   敵フェイズでsharedのAIを順に実行し、行動をアニメーション付きで再生。フェイズ表示、ターンカウント、勝敗判定によるクリア/ゲームオーバー画面。

4. **セーブ/ロード**(runtime-devに委任)
   中断セーブ(BattleState+シード+乱数消費数のシリアライズ)。ロード後に続きから同一結果になることをテストで保証。

5. **ヘッドレス自動プレイテスト**(test-engineerに委任)
   サンプル章を「自軍もAIで操作」して最後まで進行させ、クリアまたは敗北に必ず到達することを検証する `playthrough.spec.ts` を作成。100ターンで終わらなければ失敗とする。

6. **スクリーンショット回帰の導入**(test-engineerに委任)
   タイトル/マップ/行動メニュー/戦闘予測の4画面についてピクセル比較(閾値0.1%)を整備。

7. **初回バランス計測**(qa-playtesterに委任)
   サンプル章をシード100種でシミュレーションし、勝率・平均ターン・デッドロック率のベースラインレポートを `docs/reports/M2-baseline.md` に残す。

## 完了条件
- `pnpm dev:editor` なしでもサンプル章が単体起動でプレイ可能
- playthrough・スクショ回帰を含む全テストグリーン、code-reviewer全タスクAPPROVE
